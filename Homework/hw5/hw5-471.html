3. Extra Credit Tasks

Complete as many of the below tasks as you wish; you do not need to complete these in numerical order. For each task, list all sources/websites used and students consulted; you should be able to find many resources on the web about these utilities.

  1. Task 1: netcat

    Use the tool netcat (it might use the name nc in your Linux distribution) to copy a file from your local computer to a remote computer. Also use netcat to perform functionality similar to a one-to-one chat program; i.e., allow two hosts to connect and exchange messages. Many resources exist explaining netcat, please make use of these.

    Deliverable: Provide a list of commands used, resources used, and a general explanation of how the file copy and one-to-one chat were accomplished.

    You may not distribute/post this assignment without written permission from Dr. Mike Turi, Electrical and Computer Engineering Department, California State University, Fullerton.

  2. Task 2: Tunneling

    A common task in networking is to encapsulate a particular type of traffic into another type of traffic. For this exercise, you must make use of either ssh, or some other tool, to create a tunneled proxy. You will then use this proxy for your browser. In other words, the web browser traffic will travel from your PC to a remote server encapsulated in ssh traffic (or some other kind of traffic), where it will then be realized as http requests. It should resemble the flow as illustrated below. You should be able to find many resources on the web on how to accomplish this.

    Client —(ssh traffic)—> Remote SSH server —(http traffic)—> Web Server

    Client <—(ssh traffic)— Remote SSH server <—(http traffic)— Web Server

    Deliverable: You must provide a brief write up of what resources you used, all shell commands employed, and applications employed. Also include screenshots of packet captures from a successful http request.

    You may not distribute/post this assignment without written permission from Dr. Mike Turi, Electrical and Computer Engineering Department, California State University, Fullerton.

  3. Task 3: Other networking tool

    Pick any networking tool of your choice, which we have not used in a prior homework assignment, and write a brief tutorial on how you would make that tool perform a particular network task. Use common sense in what constitutes a network task. In other words, hacking another computer is not a valid network task for this class and will receive no credit (but instead a referral to academic affairs). Regardless, this tutorial should be sufficient that someone with only basic knowledge could pick up this tutorial and accomplish the given task. The task should be straightforward, similar in spirit to tasks 1 and 2. Some applications that you might consider are: scp, nmap, snort, ns-3, or anything you might choose that is prudent and which has a significant network aspect.

    Deliverable: The tutorial, a listing of all references, and any auxiliary files necessary.
